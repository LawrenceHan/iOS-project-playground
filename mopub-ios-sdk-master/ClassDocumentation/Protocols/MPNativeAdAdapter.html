MPNativeAdAdapter Protocol Reference

Conforms to NSObject
Declared in MPNativeAdAdapter.h

Overview

The MPNativeAdAdapter protocol allows the MoPub SDK to interact with native ad objects obtained from third-party ad networks. An object that adopts this protocol acts as a wrapper for a native ad object, translating its proprietary interface into a common one that the MoPub SDK can understand.

An object that adopts this protocol must implement the properties property to specify a dictionary of assets, such as title and text, that should be rendered as part of a native ad. When possible, you should place values in the returned dictionary such that they correspond to the pre-defined keys in the MPNativeAdConstants header file.

An adopting object must additionally implement -displayContentForURL:rootViewController:completion: to supply the behavior that should occur when the user interacts with the ad.

Optional methods of the protocol allow the adopting object to define when and how impressions and interactions should be tracked.

Properties

defaultActionURL

The default click-through URL for the ad.

@property (nonatomic, readonly) NSURL *defaultActionURL

Discussion

This may safely be set to nil if your network doesn’t expose this value (for example, it may only provide a method to handle a click, lacking another for retrieving the URL itself).

delegate

The MPNativeAdAdapterDelegate to send messages to as events occur.

@property (nonatomic, weak) id<MPNativeAdAdapterDelegate> delegate

Discussion

The delegate object defines several methods that you should call in order to inform MoPub of interactions with the ad. This delegate needs to be implemented if third party impression and/or click tracking is enabled.

displayContentForURL:rootViewController:

Tells the object to open the specified URL using an appropriate mechanism.

- (void)displayContentForURL:(NSURL *)URL rootViewController:(UIViewController *)controller

Parameters

URL

The URL to be opened.

controller

The view controller that should be used to present the modal view controller.

Your implementation of this method should either forward the request to the underlying third-party ad object (if it has built-in support for handling ad interactions), or open an in-application modal web browser or a modal App Store controller.

enableThirdPartyClickTracking

Determines whether MPNativeAd should track clicks

- (BOOL)enableThirdPartyClickTracking

Discussion

If not implemented, this will be assumed to return NO, and MPNativeAd will track clicks. If this returns YES, then MPNativeAd will defer to the MPNativeAdAdapterDelegate callbacks to track clicks.
